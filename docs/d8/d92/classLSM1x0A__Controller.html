Detailed Description

Main class for coordinating the LSM100A/LSM110A module.

This class provides a simple interface for the user, abstracting the underlying UART driver and AT parser. It allows initializing communication, managing event callbacks, and sending general commands obtaining their responses easily.

◆ recoverModuleConfig()

bool LSM1x0A_Controller::recoverModuleConfig ( )
private

Recovers the module configuration after an unexpected reset, re-applying the saved parameters. This is useful to maintain module state consistency even if communication is lost or the module resets itself.

Returns
true if configuration was recovered (e.g. re-applied key parameters like DevEUI, AppKey, Band, Class, etc.) and the module responded to configuration commands, false if not (e.g. did not respond to configuration commands or there was a critical error).

◆ recoverModuleState()

Returns

Sends an AT command and captures its response in a text block. Ideal for getters (e.g. "AT+DEUI=?", "AT+BAT=?").

Parameters
cmdThe exact AT command.
outBufferThe buffer where the user wants to store the response (without echo command or OK).
outSizeCapacity of outBuffer.
expectedTagIf the module is expected to prefix the response with a tag (e.g. "APP_VERSION:" or "DevEui:"), this filters it. Use nullptr for raw string.
timeoutMsWait time.
retriesNumber of retries in case of temporary error or timeout (default _maxRetries).
Returns
AtError::OK if completed and something was copied to outBuffer.
